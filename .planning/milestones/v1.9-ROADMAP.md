# Milestone v1.9: Observing Guides

**Status:** ✅ SHIPPED 2026-05-26
**Phases:** 31-34
**Total Plans:** 7

## Overview

Bring SAC's astronomy observing guides to the website as interactive, filterable pages with admin management. Phase 35 (PDF Export) was dropped at milestone close — the interactive web page covers the use case.

## Phases

### Phase 31: Object Catalog & Storage

**Goal**: The system has a searchable deep-sky object catalog and an S3 storage layer ready for guide data
**Depends on**: Nothing (first phase of v1.9)
**Requirements**: DATA-01, DATA-02, DATA-03
**Plans:** 2/2 plans complete

Plans:
- [x] 31-01-PLAN.md — OpenNGC catalog build script with Spanish names and search utilities
- [x] 31-02-PLAN.md — S3 guide storage layer (CRUD following articles-s3 pattern)

**Success Criteria:**
1. JSON catalog from OpenNGC CSV with RA/Dec, magnitude, angular size, object type, and cross-references
2. Spanish names from Stellarium translations for objects with common names
3. Guide JSON read/write to S3 with index file, same pattern as articles

## Milestone Summary

**Dropped at close:**
- Phase 35: PDF Export — branded PDF download per guide (PUB-05 deferred; interactive web page sufficient)

**Key Decisions:**
- Object catalog: OpenNGC CSV + Stellarium Spanish names; curated subset (1,101 objects) for production performance
- Storage: JSON in S3, same pattern as articles (`guides/{slug}.json` with index)
- SkyView: NASA thumbnail API, URL-based (no API key)
- Guide slug uses NFD normalization + timestamp
- Public API reuses `lib/guides.js`; ISR caching on `/guides` page
- Admin photo upload: file-only (no camera), 5MB limit, 1024px max crop dimension

**Issues Resolved:**
- Vercel bundle size: static JSON imports + curated catalog instead of full 13K object file
- Catalog search: exact ID lookup fallback, supplementary catalog for famous missing objects

**Technical Debt:**
- Phase 34 planning artifacts incomplete (no SUMMARY files despite shipped code)
- Full 13K OpenNGC catalog available locally but not used in production bundle

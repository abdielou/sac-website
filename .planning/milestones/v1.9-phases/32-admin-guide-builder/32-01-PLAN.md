<task type="auto">
  <name>Task 1: Guide business logic and permissions setup</name>
  <files>lib/guides.js, lib/permissions.js, components/admin/AdminSidebar.js</files>
  <action>
1. Create `lib/guides.js` — a business logic layer wrapping `lib/guides-s3.js` (same pattern as `lib/articles.js` wrapping `lib/articles-s3.js`). Exports:

- `listGuides()` — calls `getGuideIndex()`, returns `{ guides: [...] }` sorted by updatedAt descending. Each index entry: `{ slug, title, type, status, publishedAt, updatedAt, entryCount }`.
- `getGuide(slug)` — calls `getGuideJSON(slug)`, returns full guide object.
- `createGuide(data)` — validates required fields (title, type), generates slug from title using `slugify` (lowercase, remove accents, replace spaces with hyphens, append short timestamp suffix for uniqueness like `-1711500000`), builds guide JSON `{ title, type, slug, status: 'draft', publishedAt: null, updatedAt: new Date().toISOString(), author: data.author || null, entries: data.entries || [] }`, calls `putGuideJSON(slug, guideData)`, updates index via `getGuideIndex()` + add entry + `putGuideIndex()`, returns the created guide.
- `updateGuide(slug, updates)` — calls `getGuideJSON(slug)`, merges updates (only allowed fields: title, type, status, entries, author), sets `updatedAt`, if status changed to 'published' and no publishedAt set it now, calls `putGuideJSON()`, updates index entry, returns updated guide.
- `deleteGuide(slug)` — calls `deleteGuideJSON(slug)`, removes from index, returns `{ deleted: true }`.

For slug generation: use a simple function (no external dependency). Remove diacritics with `normalize('NFD').replace(/[\u0300-\u036f]/g, '')`, lowercase, replace non-alphanumeric with hyphens, collapse multiple hyphens, trim hyphens, append `-${Date.now()}` for uniqueness.

2. Update `lib/permissions.js`:
- Add to `Actions`: `CREATE_GUIDE: 'create_guide'`, `EDIT_GUIDE: 'edit_guide'`, `PUBLISH_GUIDE: 'publish_guide'`, `DELETE_GUIDE: 'delete_guide'`
- Add `'guides'` to `AVAILABLE_FEATURES` array
- Add guide actions to `ROLE_PERMISSIONS[Role.ADMIN]` array (admin gets all guide permissions)
- Add guide actions to `ROLE_PERMISSIONS[Role.BLOG_ADMIN]` array (blog admin can also manage guides — same content management role)

3. Update `components/admin/AdminSidebar.js`:
- Add a new nav item for guides after the articles entry: `{ href: '/admin/guides', label: 'Guias', feature: 'guides', icon: ... }`. Use a telescope/binoculars-style SVG icon (a simple eye or star icon from Heroicons).

1. Create `app/api/admin/catalog/search/route.js` — GET endpoint for catalog search:
- Auth check: `import { auth } from '../../../../../auth'` (5 levels up from api/admin/catalog/search/)
- Query params: `q` (search text, required), `type` (object type filter, optional), `limit` (max results, default 20, cap at 50)
- Import `searchCatalog` from `@/lib/catalog`
- Call `searchCatalog(q, { type, limit })` and return results as JSON
- If no `q` param, return 400 with `{ error: 'Parametro de busqueda requerido' }`
- No permission check needed — any authenticated admin can search the catalog (read-only)

2. Create `app/api/admin/guides/route.js` — GET (list) and POST (create):
- Auth via `import { auth } from '../../../../auth'`
- Permissions via `import { checkPermission } from '../../../../lib/api-permissions'` and `import { Actions } from '../../../../lib/permissions'`
- **GET**: No permission check beyond auth (any admin can view). Call `listGuides()`. Optional query param `status` to filter by draft/published. Return `{ guides: [...] }`.
- **POST**: Check `Actions.CREATE_GUIDE` permission. Parse body: `{ title, type, entries, author }`. Validate title is non-empty string, type is one of `['galaxies', 'objects']`. Call `createGuide(data)`. Return 201 with `{ guide }`.

3. Create `app/api/admin/guides/[slug]/route.js` — GET, PUT, DELETE:
- Auth via `import { auth } from '../../../../../auth'`
- Slug from `params.slug` (single segment, NOT `[...slug]` — guides use simple slugs unlike articles)
- **GET**: No extra permission check. Call `getGuide(slug)`. Return `{ guide }`. Handle not-found errors returning 404.
- **PUT**: Check `Actions.EDIT_GUIDE`. If body contains `status: 'published'`, also check `Actions.PUBLISH_GUIDE`. Parse body, call `updateGuide(slug, updates)`. Return `{ guide }`.
- **DELETE**: Check `Actions.DELETE_GUIDE`. Call `deleteGuide(slug)`. Return `{ deleted: true, slug }`.

All error handling follows existing pattern: Spanish user messages, English technical details, try/catch wrapping, console.error for logging.

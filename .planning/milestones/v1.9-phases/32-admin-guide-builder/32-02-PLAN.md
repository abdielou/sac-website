**1. `components/admin/CatalogSearch.js`** — Catalog search panel component:
- `'use client'` component accepting `onAddObject` callback prop
- Text input for search query with debounce (300ms)
- Optional object type filter dropdown (values from getCatalogStats objectTypes or hardcode common ones: Galaxy, Open Cluster, Globular Cluster, Planetary Nebula, Bright Nebula, etc.)
- Fetches `/api/admin/catalog/search?q=${query}&type=${type}&limit=20` on input
- Displays results in a compact list showing: display name (commonNameEs or commonName or name), catalog IDs (M/NGC/IC), objectType, magnitude
- Each result has an "Agregar" (Add) button that calls `onAddObject(catalogObject)`
- Shows "Buscando..." during loading, "Sin resultados" when empty
- Tailwind dark-mode styling consistent with admin dashboard

**2. `components/admin/GuideObjectRow.js`** — Single object row in guide editor:
- Props: `entry` (the guide entry with objectId + annotations + resolved object data), `index`, `total`, `onUpdate`, `onRemove`, `onMoveUp`, `onMoveDown`
- Shows object display name (resolved from catalog), catalog IDs, objectType
- Annotation fields as form controls:
  - Difficulty: `<select>` with options facil/intermedio/retante (display with accents: Facil, Intermedio, Retante)
  - Equipment: `<select>` with the 7 equipment values from CONTEXT
  - Location: `<select>` with ciudad/suburbios/oscuro (display: Ciudad, Suburbios, Cielo oscuro)
  - Optimal time: `<input type="text">` placeholder "Ej: 9:00 PM - 11:00 PM"
  - Notes: `<textarea>` 2 rows, placeholder "Notas sobre el objeto..."
- Up/down arrow buttons for reorder (disabled at top/bottom respectively)
- Red "X" remove button
- All changes call `onUpdate(index, updatedEntry)` immediately
- Collapsible annotation section — show object name + summary always, expand for full annotation fields. Default expanded.

**3. `components/admin/GuideEditor.js`** — Main guide editor component:
- `'use client'` component accepting optional `initialGuide` prop (null for create, guide object for edit)
- State: `title`, `type` ('galaxies' or 'objects'), `entries[]`, `isSaving`, `error`
- Each entry in state: `{ objectId, difficulty: '', equipment: '', location: '', optimalTime: '', notes: '', _catalogData: null }` where `_catalogData` is resolved catalog object for display
- Title input (text), Type select (Galaxias/Objetos)
- Split layout: left side is CatalogSearch panel (~40% width), right side is the entry list (~60% width). On mobile, stack vertically.
- `onAddObject` handler: when adding from search, check for duplicates by objectId, resolve catalog data, append to entries array
- Entries rendered as `GuideObjectRow` components with reorder/remove/update handlers
- Reorder: swap entries in array (moveUp swaps [i] and [i-1], moveDown swaps [i] and [i+1])
- Bottom action bar with buttons:
  - "Guardar borrador" — saves with status='draft', POST for new guide or PUT for existing
  - "Publicar" — saves with status='published'
  - "Despublicar" (only in edit mode when published) — PUT with status='draft'
  - "Eliminar" (only in edit mode) — DELETE with confirmation dialog
  - "Cancelar" — navigate back to /admin/guides
- After successful save, redirect to /admin/guides with router.push
- Show error banner at top if save fails
- When loading in edit mode, resolve `_catalogData` for each entry by calling `getObjectById` — but since that's a server function, instead fetch catalog search with the objectId to resolve display data. Or: store minimal display data in the guide entry itself at save time. BEST APPROACH: At save time, strip `_catalogData` from entries before sending to API. At load time in edit, for each entry, fetch `/api/admin/catalog/search?q=${entry.objectId}&limit=1` to resolve display names. Cache resolved objects in a Map to avoid duplicate fetches.

**4. `app/admin/guides/page.js`** — Guide list page:
- `'use client'` with `useSession` for permissions check
- Fetch `/api/admin/guides` on mount
- Status filter tabs: Todos / Publicados / Borradores
- Search input filtering by title (client-side)
- Table with columns: Titulo, Tipo (Galaxias/Objetos badge), Estado (Publicado/Borrador badge with green/yellow colors), Objetos (entry count), Actualizado (formatted date)
- Each row links to `/admin/guides/${slug}/edit`
- "Nueva guia" button at top-right linking to `/admin/guides/new` (only if user has create_guide permission)
- Empty state: "No hay guias creadas. Crea la primera guia de observacion."
- Follow the same pattern as `app/admin/articles/page.js` for loading, error states, and layout

**5. `app/admin/guides/new/page.js`** — Create guide page:
- Simple wrapper rendering `<GuideEditor />` with no initialGuide
- Page title "Nueva Guia de Observacion"

**6. `app/admin/guides/[slug]/edit/page.js`** — Edit guide page:
- `'use client'`, extracts slug from `useParams()`
- Fetches `/api/admin/guides/${slug}` on mount
- Shows loading skeleton while fetching
- Renders `<GuideEditor initialGuide={guide} />` once loaded
- Shows 404 message if guide not found
- Page title "Editar Guia"

<task type="auto">
  <name>Task 1: Public guides API route</name>
  <files>app/api/guides/public/route.js</files>
  <action>
Create GET /api/guides/public — a public (no auth) API endpoint that returns published guides grouped by type with resolved catalog data.

Logic:
1. Call `listGuides()` from `@/lib/guides`
2. Filter to only `status === 'published'` guides
3. Group by `type` ("galaxies" and "objects")
4. For each type, sort guides by `publishedAt` descending (most recent first)
5. For the response, return a structure like:
```json
{
  "galaxies": [
    {
      "slug": "...",
      "title": "...",
      "publishedAt": "...",
      "entryCount": 5
    }
  ],
  "objects": [...]
}
```
6. Accept optional query param `?slug=xxx` — when provided, fetch the full guide via `getGuide(slug)`, verify it is published (return 404 if not), resolve each entry's catalog data via `getObjectById(entry.objectId)`, and return:
```json
{
  "guide": {
    "title": "...",
    "type": "...",
    "slug": "...",
    "publishedAt": "...",
    "entries": [
      {
        "objectId": "NGC0224",
        "difficulty": "facil",
        "equipment": "telescopio_grande",
        "location": "oscuro",
        "optimalTime": "21:00",
        "notes": "...",
        "catalog": {
          "id": "NGC0224",
          "name": "NGC 224",
          "commonName": "Andromeda Galaxy",
          "commonNameEs": "Galaxia de Andromeda",
          "ra": 10.68,
          "dec": 41.27,
          "magnitude": 3.44,
          "angularSize": 178.0,
          "objectType": "Galaxy",
          "constellation": "And",
          "messier": "M 31",
          "catalogIds": { "ngc": "NGC 224", "messier": "M 31" }
        }
      }
    ]
  }
}
```

7. If `getObjectById` returns null for an entry, still include it but set `catalog: null`
8. No authentication required — this is a public endpoint
9. Spanish error messages following project convention: `{ error: 'Guia no encontrada', details: '...' }`
10. Handle S3 not configured gracefully (return empty arrays)

Import from `@/lib/guides` (listGuides, getGuide) and `@/lib/catalog` (getObjectById). Do NOT import from guides-s3 directly — use the business logic layer.

<objective>
UI implementation: `Subir foto` action + file-only upload modal + crop step.
</objective>

<tasks>

1. MemberActions: add/route "Subir foto"
  - Locate the component that renders the admin members grid row actions (existing `MemberActions` dropdown).
  - Add a new action entry labeled **“Subir foto”**.
  - When clicked, open a new modal component dedicated to admin member photo upload.

2. Admin upload modal: file-only + crop
  - Create a modal wrapper component for Phase 34 that:
    - Uses a visible file picker (accept `image/*`)
    - **Does not** offer camera mode or call `navigator.mediaDevices.getUserMedia`
    - On file selection: reads file as Data URL, opens `CropModal`
    - Uses `CropModal` with `aspect=1` (default) and waits for confirm
    - On crop confirm: calls an `onPhotoCropped(blob)` handler passed from the parent
    - Handles cancel by closing modal and clearing staged crop state

3. State & async UX
  - Disable crop confirm while cropping is generating the Blob (if needed).
  - Ensure modal closes only after user confirm (not after just file selection).

</tasks>

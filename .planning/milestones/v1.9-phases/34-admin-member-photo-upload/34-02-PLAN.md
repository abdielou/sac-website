<objective>
Back-end integration: upload to Drive, update Sheets, refresh UI.
</objective>

<tasks>

1. Server API route
  - Create/extend an API route accessible from admin UI that:
    - Accepts: member identifier (email is preferred because `uploadPhoto()` file naming uses email), and cropped image as binary (Blob -> formdata or base64) plus MIME type.
    - Calls `uploadPhoto(email, imageBuffer, mimeType)` from `lib/google-drive.js`.
    - Calls existing Sheets utilities from `lib/google-sheets.js` to update the member row’s `photoFileId`.
    - Returns JSON with at minimum `{ photoFileId }`.

2. Admin client wiring
  - In the admin modal, after crop confirm, call the API route with the selected member + image blob.
  - Show a loading state; disable confirm while saving.
  - On success:
    - Close modal
    - Trigger a refresh of the admin members grid data (reuse existing query invalidation / refetch patterns).
  - On failure:
    - Keep modal open
    - Show error message (Spanish)

3. Grid refresh correctness
  - Ensure the data refresh path uses the same field that the admin grid renders for photo presence (photoFileId or derived “missing photo” indicator).

</tasks>

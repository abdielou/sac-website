<task type="auto" tdd="true">
  <name>Task 1: Build catalog generation script and produce JSON catalog</name>
  <files>scripts/build-catalog.mjs, data/catalog/openngc.json, data/catalog/spanish-names.json, test/catalog.test.js</files>
  <behavior>
    - Test: catalog JSON file exists and is valid JSON array with >10000 entries
    - Test: each object has required fields (name, ra, dec, type, constellation) and none are null for these core fields
    - Test: Messier objects have messier cross-reference (e.g., M31 maps to NGC 224)
    - Test: objects with Spanish common names have commonNameEs populated (e.g., NGC 224 has "Galaxia de Andromeda" or similar)
    - Test: objects without common names have commonNameEs as null (not undefined or empty string)
  </behavior>
  <action>
    1. Create `scripts/build-catalog.mjs` — a Node.js script (ESM) that:
       a. Downloads OpenNGC CSV from https://raw.githubusercontent.com/mattiaverga/OpenNGC/master/database/NGC.csv
       b. Downloads common names from https://raw.githubusercontent.com/mattiaverga/OpenNGC/master/database/NGC_common_names.csv (if it exists; fallback: parse common_names from the main CSV)
       c. Parses each row into the unified object shape:
          ```
          {
            id: "NGC0224",          // primary catalog ID (padded)
            name: "NGC 224",        // display name
            catalogIds: {
              ngc: "NGC 224",       // or null
              ic: null,             // or "IC XXXX"
              messier: "M 31"       // or null
            },
            ra: 10.6847,            // RA in decimal degrees (convert from HMS if needed)
            dec: 41.2687,           // Dec in decimal degrees (convert from DMS if needed)
            magnitude: 3.44,        // V magnitude (null if unknown)
            angularSize: {
              major: 178.0,         // arcminutes (null if unknown)
              minor: 63.0           // arcminutes (null if unknown)
            },
            objectType: "Galaxy",   // human-readable type mapped from OpenNGC codes
            constellation: "And",   // 3-letter IAU abbreviation
            commonName: "Andromeda Galaxy",   // English common name or null
            commonNameEs: "Galaxia de Andromeda"  // Spanish name or null
          }
          ```
       d. Downloads Stellarium Spanish nebula names. The relevant file is:
          https://raw.githubusercontent.com/Stellarium/stellarium/master/nebulae/default/translations/nebulae.es_ES.po
          OR check https://raw.githubusercontent.com/Stellarium/stellarium/master/nebulae/default/names.dat for name mappings, then https://raw.githubusercontent.com/Stellarium/stellarium/master/po/stellarium-skycultures/es.po for Spanish translations.
          Parse .po files for msgid/msgstr pairs. Map English common names to Spanish.
       e. Writes merged Spanish names to `data/catalog/spanish-names.json` (a simple {englishName: spanishName} mapping for reference)
       f. Writes the full catalog to `data/catalog/openngc.json`

    2. OpenNGC type code mapping (column "Type" in CSV):
       - "*" -> "Star", "**" -> "Double Star", "*Ass" -> "Association"
       - "OCl" -> "Open Cluster", "GCl" -> "Globular Cluster", "Cl+N" -> "Cluster + Nebula"
       - "GGroup" -> "Galaxy Group", "GPair" -> "Galaxy Pair", "GTrpl" -> "Galaxy Triplet"
       - "G" -> "Galaxy", "PN" -> "Planetary Nebula", "HII" -> "HII Region"
       - "DrkN" -> "Dark Nebula", "EmN" -> "Emission Nebula", "Neb" -> "Nebula", "RfN" -> "Reflection Nebula"
       - "SNR" -> "Supernova Remnant", "Nova" -> "Nova", "NonEx" -> "Nonexistent"
       - "Dup" -> "Duplicate", "Other" -> "Other"

    3. RA conversion: OpenNGC stores RA as HH:MM:SS.s — convert to decimal degrees (hours * 15 + minutes/4 + seconds/240)
       Dec conversion: OpenNGC stores Dec as +DD:MM:SS.s — convert to decimal degrees

    4. Run the script: `node scripts/build-catalog.mjs` and commit the generated JSON files to the repo.

    5. Create `test/catalog.test.js` with the behavior tests above, reading from the generated JSON.

    Note: Use `fetch` (available in Node 18+) for downloads. Use no external CSV library — OpenNGC CSV is simple enough to parse with split(';') as it uses semicolons as delimiters.
  </action>
  <verify>
    <automated>node scripts/build-catalog.mjs && npx jest test/catalog.test.js --no-cache</automated>
  </verify>
  <done>data/catalog/openngc.json exists with 13K+ objects, each having RA/Dec/type/constellation. Spanish names populated for well-known objects (M31, M42, etc.). Tests pass.</done>

<task type="auto">
  <name>Task 2: Create catalog search utility library</name>
  <files>lib/catalog.js</files>
  <action>
    Create `lib/catalog.js` that provides search and lookup utilities over the generated catalog:

    1. `loadCatalog()` — reads and caches `data/catalog/openngc.json` (lazy load, singleton pattern). Since this is a ~2-3MB JSON file, load once and keep in memory.

    2. `searchCatalog(query, options)` — search by text query:
       - Matches against: name, catalogIds.ngc, catalogIds.ic, catalogIds.messier, commonName, commonNameEs
       - `options.type` — filter by objectType (e.g., "Galaxy", "Open Cluster")
       - `options.limit` — max results (default 50)
       - Case-insensitive matching
       - Exact catalog ID matches (e.g., "M31", "NGC 224") should rank first, then partial name matches
       - Strip spaces for catalog matching: "m31" matches "M 31", "ngc224" matches "NGC 224"
       - Return array of matching objects

    3. `getObjectById(id)` — look up by primary catalog ID (e.g., "NGC0224"). Returns single object or null.

    4. `getCatalogStats()` — returns { totalObjects, byType: { Galaxy: N, ... }, withSpanishNames: N }

    Export all functions as named exports (ESM-compatible CommonJS using module.exports for Next.js 12 compatibility — check how lib/articles-s3.js exports).

    Note: Since this is Next.js 12 (not App Router for the lib layer), use CommonJS require/module.exports pattern matching the existing `lib/articles-s3.js` style. However, `lib/articles-s3.js` uses ESM imports — follow that pattern (import/export).
  </action>

  </verify>

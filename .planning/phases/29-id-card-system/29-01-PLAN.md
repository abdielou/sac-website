<task type="auto">
  <name>Task 1: Install dependencies and create PDF template + QR utility</name>
  <files>
    next.config.js
    lib/id-card/IdCardDocument.js
    lib/id-card/generateIdCard.js
  </files>
  <action>
    1. Install dependencies:
       ```
       npm install @react-pdf/renderer qrcode
       ```

    2. Update `next.config.js` to add `serverExternalPackages: ['@react-pdf/renderer']` at the top level of the config object (NOT inside experimental). This prevents server-side bundling issues with @react-pdf/renderer in Next.js route handlers.

    3. Create `lib/id-card/IdCardDocument.js` — the @react-pdf/renderer PDF template:
       - Import Document, Page, View, Text, Image, StyleSheet from '@react-pdf/renderer'
       - Use CR80 card dimensions: 3.375 * 72 = 243pt width, 2.125 * 72 = 153pt height
       - Layout: SAC logo at top (use `process.cwd() + '/public/static/images/sac-white-logo.png'` for the logo path — @react-pdf/renderer runs server-side and needs absolute file paths, NOT URL paths), circular member photo below, member name, "Miembro desde {memberSince}", "Vigencia {year}", QR code at bottom
       - Background: solid #221E5A (the SAC purple). Support optional background image via `backgroundPath` prop — if an `id-bg-{year}.png` exists in public/static/images, use it; otherwise just solid color
       - Style with StyleSheet.create() — white text, centered layout
       - Export function `generateIdCardDocument({ member, photoBase64, qrDataUrl, year, logoPath, backgroundPath })` that returns JSX `<Document>` element
       - IMPORTANT: Use PNG logo only (sac-white-logo.png), NOT the SVG version — @react-pdf/renderer Image component does not support SVG

    4. Create `lib/id-card/generateIdCard.js` — the orchestrator:
       - Export `generateQrDataUrl(url)` — uses `qrcode` package's `toDataURL()` with width:200, margin:1, errorCorrectionLevel:'M'
       - Export `generateIdCardPdf(member)` — orchestrates the full pipeline:
         a. Determine current year from member.expirationDate (extract year) or fallback to current year
         b. Build verification URL: `https://sociedadastronomia.com/verify/${encodeURIComponent(member.email)}`
         c. Generate QR data URL from verification URL
         d. If member.photoFileId exists, fetch photo via `getPhoto(member.photoFileId)` from lib/google-drive.js and convert to base64 data URL (`data:${photo.mimeType};base64,${photo.buffer.toString('base64')}`)
         e. Resolve logo path: `path.join(process.cwd(), 'public/static/images/sac-white-logo.png')`
         f. Check for year-specific background: `path.join(process.cwd(), 'public/static/images/id-bg-${year}.png')` — use fs.existsSync, pass path if exists, null if not
         g. Call `generateIdCardDocument()` with all data
         h. Call `renderToStream()` from @react-pdf/renderer on the document
         i. Return the stream
       - Await ALL async operations (photo fetch, QR generation) before calling renderToStream — do NOT start rendering until all data is ready (prevents race conditions per research pitfall #2)
  </action>
  <verify>
    - `npm ls @react-pdf/renderer` shows installed
    - `npm ls qrcode` shows installed
    - `next.config.js` contains `serverExternalPackages: ['@react-pdf/renderer']`
    - `lib/id-card/IdCardDocument.js` exports `generateIdCardDocument`
    - `lib/id-card/generateIdCard.js` exports `generateIdCardPdf` and `generateQrDataUrl`
  </verify>
  <done>PDF template uses CR80 dimensions with @react-pdf/renderer primitives, QR utility generates base64 PNG data URLs, orchestrator fetches photo + QR and returns a PDF stream</done>
</task>

       - Auth check: if `!req.auth` return 401 with Spanish error "No autenticado"
       - Admin check: verify `req.auth.user.isAdmin === true`, return 403 "Acceso denegado" if not
       - Extract `email` from `params` (await params per Next.js 16 pattern): `const { email } = await params; const decodedEmail = decodeURIComponent(email)`
       - Fetch member via `getMemberByEmail(decodedEmail)` from lib/google-sheets.js

       - Call `generateIdCardPdf(member)` from lib/id-card/generateIdCard.js
       - Return Response with the stream, headers:
         - `Content-Type: application/pdf`
         - `Content-Disposition: inline; filename="id-card-${decodedEmail}.pdf"`
       - Wrap in try/catch, return 500 "Error al generar tarjeta de identificacion" on failure

    2. Create `app/verify/[email]/page.js` — public verification page:
       - This is a SERVER component (no 'use client') that fetches member data at request time

       - Import `getMemberByEmail` from lib/google-sheets.js
       - Fetch member data server-side
       - If member not found: show "No se encontro informacion de membresia" message
       - If member found and status is 'active' or 'expiring-soon': show green confirmation with member name (first name + last name only, no email) and "Miembro activo de SAC" with the expiration year
       - If member found but status is 'expired' or 'applied': show yellow/orange message "Membresia no vigente" with status info
       - Include SAC logo, clean centered layout, responsive design with Tailwind
       - Export metadata: `{ title: 'Verificacion de Membresia - SAC' }`

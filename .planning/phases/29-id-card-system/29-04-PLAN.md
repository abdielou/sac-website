  <how-to-verify>
    Start the dev server: `npm run dev`

    **1. Admin PDF Generation (ID-01, ID-02, ID-03):**
    - Sign in as admin
    - Open browser to: `http://localhost:3000/api/admin/members/{a-member-email}/id-card`
    - Expected: PDF opens in browser with member name, photo (if they have one), membership year, QR code
    - Verify the QR code — scan it or note the URL. It should point to `https://sociedadastronomia.com/verify/{email}`
    - Try with a member who has no photo — should still generate with placeholder area
    - Try with a non-existent email — should return 404 JSON error

    **2. Public Verification Page (ID-02):**
    - Open: `http://localhost:3000/verify/{a-member-email}` (no login needed)
    - Expected: Shows member name and active/inactive status
    - Should NOT show email, phone, or other sensitive data
    - Try with active member — green confirmation
    - Try with expired member — warning message
    - Try with non-existent email — "not found" message

    **3. Member Profile Preview (PROF-06):**
    - Sign in as a member (not admin)
    - Go to: `http://localhost:3000/member/profile`
    - Expected: ID card preview section appears below the profile information
    - Card shows member photo, name, membership year
    - If member is active: "Descargar PDF" button works and downloads a PDF
    - If member has no photo: placeholder shown in card

    **4. Admin Missing Photo Filter (ID-05):**
    - Sign in as admin
    - Go to: `http://localhost:3000/admin/members`
    - Expected: "Sin foto" filter pill appears in the filter bar
    - Click it — only members without photos should appear
    - Enable "Foto" column via column selector — should show "Si" or "Sin foto" per row
    - Mobile: cards should show orange "Sin foto" badge for members without photos

    **5. Old Page Cleanup (CLEAN-01):**
    - Navigate to: `http://localhost:3000/id`
    - Expected: Redirects to `/member/profile`

    **6. Build Verification:**
    - Run `npm run build` — should complete without errors
